# alt-kde-fix

Скрипты для Alt Linux 11 на KDE, предназначенные для включения отключенного функционала, настройки sudo, доустановки компонентов (для версии Starterkit) и других улучшений.

> **Назначение:** Alt Linux 11 Kworkstation и Starterkit, KDE, домашнее (некоммерческое) использование.

---

## Скрипты

### `alt11-kde-fixer.sh`

Основной скрипт, выполняющий следующие действия:

| Блок | Что делает |
|------|------------|
| **sudo** | Устанавливает пакет `sudo` (если отсутствует), даёт права пользователю |
| **Группы** | Добавляет пользователя в группы `wheel`, `lp`, `dialout` (для работы периферии на USB) |
| **Пакеты** | Донастраивает Starterkit до полноценного состояния, ставит epm и другие нужные пакеты |
| **Обновление** | Обновляет систему, epm, устанавливает Flatpak с зависимосями |
| **Flatpak** | Открывает доступ к домашнему каталогу (возможность перетаскивать файлы из файлового менеджера в флэтпак-приложения) |
| **Копирование** | Исправляет индикацию копирования на внешний диск через параметр `DIRTY_BYTES` |
| **KDE темы** | Включает загрузку тем и виджетов из интернета |
| **3D превью** | Настраивает создание превью для файлов `.stl`, `.FCstd`, `.dwg` (3D проекты, AutoCAD). Для `.dwg` скачивается и устанавливается **NConvert** |

> 💡 Скрипт разделён на логические блоки с понятными разделителями. Ненужные функции можно удалить, вырезав соответствующий блок.

### `discover-proxy-fix.sh`

Исправляет ошибку **Discover**, возникающую после удаления локального прокси.

**Проблема:** Discover продолжает пытаться подключиться к локальному прокси-серверу, который был удален (например, `127.0.0.1`), что приводит к ошибке при проверке обновлений.

**Решение:** Скрипт пересоздаёт базу данных PackageKit.

> ⚠️ Требуется активное подключение к интернету.

## Использование

1. **Скачайте архив**  
   Нажмите зелёную кнопку **`<> Code`** → **Download ZIP**

2. **Распакуйте**  
   В домашний каталог (можно оставить в папке `Загрузки`)

3. **Дайте права на выполнение**  
   Правый клик по скрипту → **Свойства** → **Права** → ✅ *Разрешить выполнение файла как программы*

4. **Запустите в терминале**  
   Правый клик по скрипту → **Действия** → **Запустить в Konsole**

5. **Введите пароль**  
   При запросе пароля в терминале — символы не отображаются, но вводятся. Далее скрипт выводит комментарии по ходу выполнения.
