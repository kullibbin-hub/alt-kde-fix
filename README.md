# alt-kde-fix
Скрипты для Altlinux 11 на KDE, для включения отключенного функционала, получения sudo, доустановки компонентов (если установлен Starterkit) и т д. Предназначен для систем Altlinux 11 Kworkstation и Starterkit, с KDE, для домашнего (некоммерческого) использования.

Скрипт alt11-kde-fixer.sh - устанавливает отсутствующий по-умолчанию пакет sudo, дает пользователю права на его использование, включает пользователя в группы wheel, lp, dialout (для работы периферии по USB), устанавливает пакеты для приведения Starterkit в надлежащее состояние для полноценного использования. Обновляет систему, epm, устанавливает flatpak со всем необходимым. Включает доступ приложений flatpak к домашнему каталогу - это позволяет бросать файлы прям из файлменеджера на окно программы. Это, и другие пункты скрипта, можно удалить, в скрипте все разделено на блоки, просто удалить ненужный блок.

Далее, исправляется индикация копирования на внешний диск путем приведения DIRTY_BYTES к рекомендованным значениям. 

Включается отключенная возможность загрузки из интернета тем и виджетов для КДЕ.

Далее, настраивается создание превью для значков файлов форматов stl, FCstd, dwg - это файлы 3D проектов, в том числе и Автокада. Для извлечения превью для файлов dwg скачивается и устанавливается NConvert. 

Еще раз, в скрипте все разделено на блоки, назначение блока понятно, ненужные функции можно просто удалить из скрипта, вырезав текст от разделителя до разделителя.

Скрипт discover-proxy-fix.sh - исправляет ошибку Discover, когда был установлен локальный прокси, Discover работал через него, но после удаления прокси, Discover продолжает подключаться к его интерфейсу, которого уже нет. При этом, при проверке обновлений, он выводит ошибку в окне, где сказано про невозможность соединения с адресом 127.0.0.1 и т д.  Данный скрипт пересоздает базу данных packagekit и его работа нормализуется. Должно быть активное подключение к интернету.

## Использование. 

По зеленой кнопке "<>Code" вверху страницы выбрать Download zip, скачается архив. Распаковать в домашний каталог (можно оставить в папке "Загрузки"), в распакованной папке в свойствах скриптов по правому клику - свойства - Права - Разрешить выполнение файла как программы, поставить тут галочку. Далее, запустить скрипт в терминале: правым кликом по скрипту - Действия - Запустить в konsole. Дальше скрипт подскажет, когда потребуется ввести пароль. Символы при вводе пароля в окне терминала не отображаются, но вводятся! Далее в окне терминала выводятся комментарии по ходу его выполнения.
